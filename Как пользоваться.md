# Как пользоваться QA базой знаний iThive

## Структура

```
Obsidian/
├── 00 - Index.md              # Главная точка входа
├── Architecture.md             # Связи между модулями
├── MOC - Модули.md            # Карта всех модулей
├── MOC - Инструкции.md        # Карта всех инструкций
├── Modules/                    # Файлы модулей (16 шт)
├── Instructions/               # Инструкции (добавляет @obsidian-indexer)
├── Test Cases/                 # Тест-кейсы и чек-листы (добавляет @qaithive)
├── Templates/                  # Шаблоны документов
└── .opencode/agent/            # Субагенты
```

## 1. Добавление инструкций

### Через субагента @obsidian-indexer

1. В opencode запусти: `@obsidian-indexer`
2. Вставь текст инструкции (MD, TXT, или просто текст)
3. Субагент сам:
   - Определит модуль
   - Создаст файл в `Instructions/`
   - Добавит wikilinks, теги, properties
   - Обновит MOC-файлы

### Что происходит автоматически

- Файл получает имя: `{Module} - {Topic}.md`
- Добавляются теги: `#module/{slug}`, `#type/instruction`
- Добавляются ссылки на модуль и связанные модули
- Обновляется `MOC - Инструкции.md`
- Обновляется файл модуля в `Modules/`

## 2. Генерация чек-листов

### Через субагента @qaithive

1. В opencode запусти: `@qaithive`
2. Опиши задачу (новая фича, баг-фикс, регрессия)
3. Субагент изучит базу знаний и выдаст чек-лист

### Сохранение чек-листа

Если попросишь сохранить — субагент создаст файл в `Test Cases/` с форматом:
- Имя: `{Module} - {Topic}.md`
- Теги: `#module/{slug}`, `#type/test-case`
- Wikilinks на модуль и связанные документы

## 3. Ответы на вопросы по модулям

### Через субагента @qa-teacher

1. В opencode запусти: `@qa-teacher`
2. Задай вопрос по настройке или использованию модуля
3. Субагент ответит, опираясь на инструкции из базы знаний

## 4. Заполнение модулей

**Важно:** чем лучше заполнены файлы модулей, тем лучше AI составляет чек-листы.

В каждом файле `Modules/Module - {Name}.md` заполни:

- **Описание** — кратко что делает модуль
- **Функциональность** — список ключевых функций/сценариев

## 5. Связи между модулями

Файл `Architecture.md` содержит:
- Группы модулей (UI, Коммуникация, Контент, HR, Управление)
- Кросс-модульные зависимости

**Заполни таблицу зависимостей** — это критично для AI чтобы понимать какие модули затрагивает новая фича.

## 6. Теги

### По модулям
`#module/helpdesk`, `#module/knowledgebase`, `#module/library`, `#module/branding`, `#module/gamification`, `#module/homepage`, `#module/hints`, `#module/workplaces`, `#module/university`, `#module/multilang`, `#module/notifications`, `#module/polls`, `#module/assessment360`, `#module/goals-tree`, `#module/goals-cascade`, `#module/mediagallery`

### По типу
`#type/instruction`, `#type/test-case`, `#type/documentation`

## 7. Быстрые команды

| Действие | Команда |
|----------|---------|
| Добавить инструкцию | `@obsidian-indexer` → вставь текст |
| Сгенерировать чек-лист | `@qaithive` → опиши задачу |
| Сохранить чек-лист | `@qaithive` → попроси сохранить |
| Спросить по модулю | `@qa-teacher` → задай вопрос |
| Найти по модулю | Поиск: `#module/{slug}` |
| Найти по типу | Поиск: `#type/instruction` |
