# Геймификация - Ручные операции с баллами

**Модуль:** [[Module - Геймификация]]

## Описание

Администраторы могут производить любые операции вручную: увеличивать/уменьшать баланс, присуждать баллы «Спасибо», присуждать призовые места в рейтингах. Проведённые операции нельзя редактировать или удалять.

## Ключевые функции

- **Начисление баллов вручную** — выбор сотрудника (в т.ч. загрузкой из списка: ФИО, Email); 7 типов начисления
- **Ручное изменение счета** — увеличение баланса обычных баллов или баллов «Спасибо»; без лимита
- **Благодарность** — аналог «Сказать спасибо» через админку; выбор бейджа (категории), указание суммы
- **Призовое место в рейтинге** — ручное начисление призерам; нельзя начислить повторно в том же месяце; баллы определяют места
- **Дни рождения** — начисление в честь дня рождения
- **Стаж работы в компании** — начисление в честь годовщины работы
- **Выполнение задачи** — начисление за закрытие задачи
- **Выполнения задания бизнес-процесса** — начисление за активити в БП
- **Списание баллов вручную** — уменьшение баланса (возможен отрицательный баланс); не влияет на уровень сотрудника
- **Покупка из магазина** — списание за товар, не добавленный в магазин

## Предусловия

- Права на начисление/списание баллов (по умолчанию — группа «Администраторы»)
- Доступ к разделу «Операции»

## Шаги (начисление с загрузкой из списка)

1. Перейдите в раздел «Операции» → нажмите «Добавить»
2. Нажмите «Загрузить из списка»
3. Вставьте список сотрудников в одном из форматов: Имя Фамилия, Фамилия Имя, Email, ФИО
4. Выберите «Формат искомого значения»
5. Нажмите «Далее» → проверьте найденных пользователей → «Подтвердить»
6. Выберите тип операции и тип начисления
7. Укажите сумму, валюту, описание → «Далее»
8. Распределите баллы по каждому сотруднику → «Выполнить»

## Ожидаемый результат

Баланс указанных сотрудников обновлён, операции отобразились в разделе «Операции».

## Граничные случаи

- Операции нельзя редактировать или удалять — только создавать новые
- При загрузке из списка: если формат данных не совпадает — сотрудник не будет найден
- Причины ненайденных: уволен, не привязан к подразделению, опечатка, неверный формат
- Повторяющиеся сотрудники не дублируются в поле «Кому»
- Призовое место: в одном месяце можно только один раз наградить призеров каждого рейтинга
- Списание баллов не влияет на уровень сотрудника (ранг не понижается)
