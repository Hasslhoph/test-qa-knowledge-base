# Создание и редактирование целей

**Модуль:** [[Module - Управление целями. Каскад]]

## Описание

Инструкция по созданию типов целей, добавлению, редактированию и удалению целей в каталоге модуля.

## Ключевые функции

> Управление каталогом корпоративных целей.

- Создание типов целей через административную часть Битрикс24
- Добавление целей с настройками: активность, тип, вес, каскадирование, цвет, права
- Редактирование и удаление целей
- Ограничения при использовании целей в запущенных процессах

## Создание типов целей

Типы целей создаются в административной части Битрикс24 (только администраторы портала):

1. Перейти: `адрес_портала/bitrix/admin/`
2. «Контент» > «Управление целями — Каталог целей»
3. «Настройки информационного блока»
4. Вкладка «Свойства» → «Тип цели» → три точки (...)
5. Таблица «Значения списка» → добавить типы целей

## Добавление целей

Доступ только администраторам модуля. Раздел «Каталог целей» → «Добавить цель».

### Параметры цели

1. **Активность цели** — влияет на видимость в карточке целей
2. **Название цели** — обязательное поле
3. **Тип цели** — категория (стратегические, маркетинговые и т.д.)
4. **Минимальный вес цели** — минимальный приоритет (по умолчанию 10)
5. **Шаг изменения веса цели** — шаг для указания веса
6. **Максимальный % выполнения** — максимальный прогресс (например, 120% для перевыполнения)
7. **Шаг изменения % выполнения** — шаг для указания прогресса
8. **Текст подсказки при редактировании веса** — пояснение для сотрудников
9. **Текст подсказки при редактировании % выполнения** — пояснение для сотрудников
10. **Возможности каскадирования** — вкл/выкл автоматическое каскадирование
11. **Фоновый цвет** — визуальное разделение целей в карточках
12. **Настройка прав** — группы пользователей с доступом к цели

## Редактирование и удаление целей

- **Редактирование:** бургер-меню (≡) → «Изменить»
- **Удаление:** бургер-меню (≡) → «Удалить»

### Важные ограничения

- Если цель используется в запущенном процессе — тип и название редактировать нельзя
- Отключение активности не скрывает цель из уже добавленных карточек
- Удаление используемой цели — она исчезнет из карточек и сводного отчета (не рекомендуется)

## Предусловия

- Роль администратора портала (для создания типов)
- Роль администратора модуля (для добавления целей)

## Граничные случаи

- Суммарный вес всех целей в одной карточке должен быть равен 100
- Минимальный вес нужен для реалистичной расстановки приоритетов
- При включенном каскадировании руководители заполняют карточки первыми
