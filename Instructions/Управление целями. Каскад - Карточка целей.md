# Карточка целей

**Модуль:** [[Module - Управление целями. Каскад]]

## Описание

Карточка целей — основной инструмент для постановки целей, каскадирования и корректировки критериев. Создается автоматически при запуске процесса. В течение одного процесса у сотрудника может быть только одна карточка целей.

## Ключевые функции

> Управление целями сотрудника на процесс.

- Автоматическое создание при запуске процесса
- 6 статусов: Составление → Согласует руководитель → Согласует функциональный руководитель → Согласует директор → Утверждает администратор → Утверждено
- Добавление корпоративных и индивидуальных целей
- Каскадирование целей подчиненным
- Делегирование заполнения карточки
- Корректировка комментария (при включенной опции)
- Экспорт списка целей (Excel, печать, почта)

## Обзор карточки целей

### Статусы карточки

1. **Составление** — постановка целей, каскадирование, указание критериев и веса
2. **Согласует руководитель** — проверка прямым руководителем
3. **Согласует функциональный руководитель** — проверка функциональным руководителем
4. **Согласует директор** — проверка руководителем на уровень выше
5. **Утверждает администратор** — проверка и утверждение администратором модуля
6. **Утверждено** — цели, критерии и вес нельзя редактировать

### Информация о карточке

- **Ответственный** — пользователь, для которого создана карточка
- **Делегат** — делегирование заполнения другому сотруднику (только на этапе заполнения)
- **Данные о процессе** — дедлайны, сроки процесса, корректировки
- **Прогресс по цели** — отображается после утверждения карточки

### Блок добавления целей

Поля при добавлении цели:
1. **Тип цели** — категория (обязательно)
2. **Цель/Задача** — конкретная цель из списка (обязательно, макс. 10 целей)
3. **Критерии выполнения** — задачи для достижения цели (обязательно)
4. **Комментарий** — вспомогательное поле
5. **Вес** — приоритетность (сумма = 100)
6. **Файл** — прикрепление файла (один файл)
7. **Каскадировать** — передача подчиненному (только руководители)
8. **Зависит от** — подразделение, влияющее на цель
9. **Связанные** — задачи и проекты

## Заполнение карточки

- Кнопка «Добавить цель» → заполнить поля → «Сохранить цели» или «Отправить на согласование»
- При включенном каскадировании руководители заполняют первыми
- Уведомления приходят всем сотрудникам с запущенным процессом

## Карточка с каскадированными целями

- Каскадированные цели недоступны для редактирования
- Можно изменить комментарий и вес (изменения сохраняются только в карточке сотрудника)

## Корректировка целей

При включенной опции временного окна:
- Редактирование поля «Комментарий» до/после отчетного периода
- От -20 до 20 рабочих дней
- Повторное согласование не требуется
- Изменения отслеживаются во вкладке «История»

## Карточка для нового сотрудника

- Автоматическое создание карточки при появлении нового сотрудника в подразделении
- Автоматическая задача на заполнение (можно отключить в настройках)

## Вкладки

- **История** — все действия, этапы согласования, комментарии
- **Оценки** — карточки оценки для заполнения прогресса

## Управление карточками (администратор)

- **Редактировать** — открыть карточку сотрудника
- **Перезапустить процесс** — вернуть в статус «Составление»
- **Уведомить** — отправить уведомление сотруднику
- **Удалить** — удалить карточку (не рекомендуется при запущенном процессе)

## Экспорт списка целей

- **В Excel** — файл xls с отображенными столбцами
- **Распечатать** — печать в табличном формате
- **Отправить на почту** — xls на почту пользователя

## Предусловия

- Запущенный процесс для подразделения
- Заполненный каталог целей с открытыми правами доступа

## Граничные случаи

- При каскадировании руководители должны заполнить карточки первыми
- Если руководители не участвуют — подчиненные заполняют сразу
- Суммарный вес всех целей = 100
- Делегат теряет доступ после утверждения карточки
- Делегирование доступно только для карточки целей (не для оценок)
