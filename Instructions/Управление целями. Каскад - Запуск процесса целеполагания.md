# Запуск процесса целеполагания

**Модуль:** [[Module - Управление целями. Каскад]]

## Описание

Инструкция по подготовке, запуску и остановке процессов целеполагания для подразделений компании.

## Ключевые функции

> Управление жизненным циклом процессов.

- Очистка тестовых данных перед «боевым» запуском
- Прикрепление процессов к подразделениям
- Запуск и остановка процессов
- Настройка автоматического перезапуска

## Подготовка перед запуском

Очистка тестовых данных (администратор портала):

1. **Карточки целей:** «Контент» > «Процессы» > «Управление целями — Карточка целей сотрудника» → удалить все записи
2. **Карточки оценок:** «Контент» > «Процессы» > «Управление целями — Карточка оценок сотрудника» → удалить все записи
3. **Цели сотрудников:** «Контент» > «Highload-блоки» > `ItHiveUserGoals` → удалить все записи
4. **Данные о прогрессе:** «Контент» > «Highload-блоки» > `ItHiveUserRating` → удалить все записи

## Прикрепление процесса к подразделению

Только администратор модуля. Раздел «Структура компании» → иконка мишени у подразделения.

### Параметры

1. **Управление целями** — выбор процесса для подразделения
2. **Исключить сотрудников** — сотрудники не участвуют в процессе
3. **Исключить руководителя** — руководитель не участвует в процессе
4. **Автоматически перезапускать процесс** — автозапуск после завершения

### Наследование настроек

Опция «Наследует настройки вышестоящего подразделения»:
- Прикрепленный процесс
- Галочка «Автоматически перезапускать процесс»

## Запуск процесса

Только администраторы модуля. Раздел «Контроль процессов» → бургер-меню (≡) → «Запустить процесс».

### Нюансы запуска

| # | Нюанс |
|---|-------|
| 1 | При запуске модуль запоминает текущую оргструктуру. Изменения в оргструктуре после запуска не влияют на согласование |
| 2 | На одного пользователя одновременно может быть запущен только 1 процесс и создана 1 карточка целей |

- Запуск нескольких процессов: выделить галочками → «Запустить процесс»
- Если до завершения процесса ≤ 20 дней — запуск невозможен

## Остановка процесса

- Бургер-меню (≡) → «Остановить процесс»
- Несколько процессов: выделить галочками → остановить
- При ручной остановке деактивируются все карточки целей, оценок и индивидуальные цели

## Предусловия

- Процессы созданы
- Процессы прикреплены к подразделениям
- Роль администратора модуля

## Граничные случаи

- При запуске фиксируется текущая оргструктура — изменения после запуска не учитываются
- Сотрудник в нескольких подразделениях — только 1 процесс на пользователя
- Запуск невозможен если до завершения периода ≤ 20 дней (настраивается)
