# Legacy. Нюансы работы с модулем

**Модуль:** [[Module - Брендирование Битрикс24]]

## Описание

Статья описывает важные нюансы работы с модулем «Брендирование» в дизайне «Legacy» (до intranet 25.900.0). Рассмотрены два способа работы с модулем, условия видимости дизайна и способы возврата к стандартным настройкам.

## Ключевые функции

### Два способа работы с модулем

1. **Настройка стилей в модуле** — ручная настройка каждого параметра (цвет, шрифт, обводка) по отдельности. Удобна для минимальных изменений.

   Элементы, которые можно задать **только через модуль**:
   - Все настройки раздела «Авторизация»
   - Стили текста (заглавные буквы и т.п.)
   - Скрыть/показать виджеты
   - Загружаемые элементы: фавикон, наборы иконок

2. **Создание дизайна в Figma** — использование интерактивного макета Улья для глобальных изменений дизайна. Позволяет оперативно визуализировать оформление, создавать копии макетов, импортировать стили через плагин.

### Условия видимости дизайна

- Модуль работает только с классическими темами: «Битрикс 24» и «Битрикс 24 с узором»
- При другой теме настройки не отображаются, кроме: раздела «Авторизация», фавикона, логотипа, наборов иконок

### Доступные шрифты

Битрикс24 поддерживает только 4 шрифта: `family-system`, `system-mono`, `helvetica`, `open-sans`

### Возврат стандартных настроек

1. **Отключить модуль** — настройки сохраняются, но не отображаются
2. **Сбросить все настройки** — полная очистка, загруженные элементы (фавикон, иконки) стираются

## Предусловия

- Модуль установлен и активен
- Пользователь имеет доступ к настройкам брендирования

## Граничные случаи

- При отключении модуля загруженные элементы (фавикон, иконки) также не отображаются
- При сбросе настроек загруженные элементы стираются — рекомендуется предварительно экспортировать настройки
- Шрифты, не входящие в список поддерживаемых Битрикс24, не будут отображаться на портале
