# Оценка 360 - Запуск оценки

**Модуль:** [[Module - Оценка 360]]

## Описание

Процесс запуска оценки для одного сотрудника или группы сотрудников. Включает настройку карточки оценки, выбор оценивающих, компетенций, открытых вопросов.

## Ключевые функции

### Запуск для одного сотрудника
- Выбрать сотрудника из списка → «Далее»
- Проверить, что все предыдущие карточки имеют статус «Оценка завершена»

### Настройка карточки оценки
1. **Кого оцениваем** — автоматически подтягивается выбранный сотрудник
2. **Руководители** — автоматически из оргструктуры (обязательное поле). Можно указать вручную. Можно отправить на согласование руководителю
3. **Коллеги** — автоматически из оргструктуры (минимум 1 по умолчанию). Максимум 25 (настраивается). Оцениваемый может пригласить коллег самостоятельно (лимит по умолчанию — 2)
4. **Подчинённые** — автоматически из оргструктуры (если сотрудник руководитель). Максимум 25
5. **Клиенты и контакты CRM** — отправка ссылки по SMS или e-mail. Оценивают только soft-компетенции
6. **Что оцениваем** — компетенции из шаблона или вручную. Можно выбрать, кто оценивает каждую компетенцию
7. **Какие вопросы задать** — открытые вопросы для респондентов. Можно выбрать существующие или создать новые. Настройка: кто отвечает, обязательность

### Запуск для группы сотрудников
- Выбрать отдел или группу пользователей → «Далее»
- Особенности: один набор компетенций для всех, нельзя привлечь клиентов из CRM
- Таблица со списком сотрудников и оценивающими ролями
- Можно удалять/добавлять сотрудников из группы

### Проверка данных перед запуском
- Слайдер с информацией: сроки, участники, компетенции, аналитика, загруженность
- Настройка срока сбора оценок (7-30 дней)
- Добавление тегов к запускаемым оценкам

### Согласование карточки
- Руководитель получает уведомление со ссылкой
- Может изменить настройки и запустить/отменить оценку

### Приглашение коллег
- Сотрудник может пригласить коллег после запуска оценки
- Лимит устанавливается в настройках модуля
- Выбор коллег нельзя изменить после сохранения

## Предусловия

- Права на запуск оценки (администраторы, HR-отдел)
- Созданы компетенции и/или шаблоны
- Нет незавершённых оценок для оцениваемых сотрудников

## Граничные случаи

- Нельзя запустить новую оценку, пока не завершена предыдущая
- Один пользователь не может быть указан в разных ролях (например, коллега и подчинённый)
- При массовом запуске нельзя привлечь клиентов из CRM
- Контакты из CRM оценивают только soft-компетенции
