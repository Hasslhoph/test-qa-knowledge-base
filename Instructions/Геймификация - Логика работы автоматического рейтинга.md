# Геймификация - Логика работы автоматического рейтинга

**Модуль:** [[Module - Геймификация]]

## Описание

Подробное описание логики работы автоматических рейтингов: за какой период учитываются действия, как определяется место сотрудника, как начисляются призовые баллы. Расчёты показаны на примере рейтинга «Эффективный исполнитель задач».

## Ключевые функции

- **Помесячный учёт** — учитываются только действия текущего месяца, 1-го числа рейтинги обнуляются
- **Критерии рейтинга «Эффективный исполнитель задач»** — задача завершена, дата завершения в текущем месяце, сотрудник — исполнитель, постановщик — руководитель, есть дедлайн
- **Параметры рейтинга** — каждый параметр имеет свой вес влияния (чем больше вес, тем больше влияние)
- **Формула расчёта показателя** — $P_i = \sum_{1}^{M_i}\left(K_1 \frac{1}{1 + N_{mi}} + K_2 \cdot L_{mi}\right)$
- **Начисление призовых баллов** — Сумма × Показатель сотрудника ÷ Сумма всех показателей за призовые места

## Предусловия

- Понимание структуры автоматических рейтингов
- Доступ к настройкам модуля для просмотра весов параметров

## Шаги

1. Изучите период учёта действий (текущий месяц, сброс 1-го числа)
2. Ознакомьтесь с критериями рейтинга
3. Разберитесь в параметрах и их весах
4. Изучите формулу расчёта показателя на примере
5. Поймите логику распределения призовых баллов

## Ожидаемый результат

Пользователь понимает, как рассчитываются позиции в автоматических рейтингах и как распределяются призовые баллы.

## Граничные случаи

- Действия по рейтингам не суммируются каждый месяц — они сбрасываются
- Список текущих участников очищается при обнулении рейтинга
- Список появится после пересчёта 2-го числа нового месяца
- При повторном начислении баллов за тот же месяц появляется запрет
- Замечания в задачах могут понижать место сотрудника в рейтинге
