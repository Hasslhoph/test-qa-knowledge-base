# Роли пользователей и права доступа

**Модуль:** [[Module - Управление целями. Каскад]]

## Описание

В модуле предусмотрено шесть ролей пользователей с различными правами доступа к разделам и функциям модуля.

## Ключевые функции

> 6 ролей с разными уровнями доступа и возможностями.

- **Сотрудник** — базовая роль, доступ только к своим карточкам
- **Руководитель подразделения** — согласование, каскадирование, просмотр подчиненных
- **Функциональный руководитель** — дополнительное согласование (опционально)
- **Директор** — согласование всех сотрудников в подчинении
- **Администратор модуля** — полный доступ, финальное утверждение
- **Сотрудник внутреннего контроля** — просмотр всех данных (только чтение)

## Роли

### Сотрудник

- Заполняет свою карточку целей и карточку оценки
- Отправляет их на согласование руководителю
- Доступ только к своим карточкам целей и оценок
- Другие разделы модуля недоступны

### Руководитель подразделения

- Первое звено в согласовании карточек целей и оценок
- Согласует цели и оценки сотрудников своего отдела и подотделов
- Может каскадировать цели подчиненным
- Заполняет карточки целей вместо подчиненных
- Заполняет свою карточку и отправляет вышестоящим руководителям

**Доступные элементы:**
- Собственные карточки целей и оценок
- Карточки подчиненных (включая подотделы)
- Сводный отчет по своему отделу
- Список процессов для подразделений в подчинении (без редактирования)

### Функциональный руководитель

- Не является обязательной ролью
- Может не являться руководителем подразделения в оргструктуре
- Согласует карточки привязанных сотрудников (после руководителя подразделения)
- Не может автоматически каскадировать цели, но может вручную добавить цель на этапе согласования

**Доступные элементы:**
- Собственные карточки (если есть)
- Карточки функциональных подчиненных
- Карточки прямых подчиненных (если есть)
- Сводный отчет по своему отделу

**Добавление роли:** Только администраторы модуля. Раздел «Настройки» > «Функ. руководитель». Слева — руководители, справа — подчиненные.

### Директор

- Руководитель руководителя согласно оргструктуре Битрикс24
- Согласует карточки всех сотрудников в подчинении
- Может каскадировать цели, заполнять карточки вместо подчиненных
- Заполняет свою карточку и отправляет на утверждение администратору

**Доступные элементы:**
- Собственные карточки
- Карточки сотрудников в подчинении
- Сводный отчет по подразделениям в подчинении
- Список процессов для подразделений в иерархии

### Администратор модуля

- Доступ ко всем функциям модуля
- Добавляет цели, создает и запускает процессы
- Финальное звено — утверждает карточки всех пользователей
- По умолчанию — группа администраторов портала

**Настройка прав:** Раздел «Настройки» > «Права». Выбор группы администраторов модуля.

### Сотрудник внутреннего контроля

- Не обязательная роль
- Помогает администратору следить за процессом
- Доступ ко всем разделам, карточкам — только в режиме просмотра
- Не может добавлять, редактировать, согласовывать данные

**Настройка прав:** Только администраторы модуля. Раздел «Настройки» > «Права».

## Предусловия

- Модуль установлен
- Администраторы портала настроили права доступа

## Граничные случаи

- Функциональный руководитель не может автоматически каскадировать цели
- Сотрудник внутреннего контроля имеет доступ только для чтения
- Права на изменение настроек доступа есть только у администраторов модуля
