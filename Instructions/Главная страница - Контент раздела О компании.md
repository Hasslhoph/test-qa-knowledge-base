# Контент раздела «О компании»

**Модуль:** [[Module - Главная страница]]

## Описание

Инструкция по созданию страницы «О компании» на втором сайте и наполнению её контентом.

## Ключевые функции

> Перечисли основные функции/сценарии. Критично для AI-генерации чек-листов.

- **Создание страницы «О компании»** на втором сайте
- **Символьный код** — уникальный адрес страницы (должен отличаться от основного сайта)
- **Привязка к подразделению** — только сотрудники подразделения видят страницу
- **9 типов блоков:** Приветственное слово, О компании, Миссия и ценности, Видео о компании, Медиабанк, Команда профессионалов, Структурные подразделения, История развития, Адреса
- **Копирование блоков** с основного сайта на второй

## Предусловия

- Права администратора портала
- Включённый пункт меню «О компании» для второго сайта

## Шаги

### Создание страницы «О компании» на втором сайте

1. `/bitrix/admin/` → «О компании» > «Управление блоками»
2. Нажмите «Добавить раздел»
3. Вкладка «Раздел»: Название (например, «Второй департамент»)
4. Вкладка «Дополнительно»: Символьный код (например, `about-dep2`, **не** `about`)
5. Вкладка «Доп. поля»: Подразделение (сотрудники второго департамента)
6. Опция «Отображать контент сотрудникам, не имеющим привязку к подразделению» (для администраторов)
7. Сохраните

### Наполнение страницы блоками

1. Создайте папку второго сайта в нужном инфоблоке (например, «О компании» > «Приветственное слово» → «Добавить раздел»)
2. Откройте папку → «Добавить элемент» → создайте контент
3. Или скопируйте блок из папки основного сайта
4. Перейдите в «Управление блоками» → откройте папку основного сайта → найдите блок → «Копировать»
5. На странице копирования:
   - «Откуда выводить данные»: выберите блок и название второго сайта
   - «Разделы»: привяжите к папке второго сайта
6. Сохраните

## Граничные случаи

> Edge cases, ограничения, известные проблемы.

- Символьный код **должен отличаться** от основного сайта (иначе всегда будет открываться раздел первого сайта)
- Без выбора подразделения страница будет недоступна никому
- Администратор должен входить в указанное подразделение или включить опцию отображения для непривязанных
