# Стилизация Главной на разных сайтах

**Модуль:** [[Module - Главная страница]]

## Описание

Инструкция по применению кастомных CSS-стилей для Главной страницы на одном или нескольких сайтах (департаментах).

## Ключевые функции

> Перечисли основные функции/сценарии. Критично для AI-генерации чек-листов.

- **Кастомные стили** для одного сайта
- **Разные стили** для каждого сайта при многосайтовости
- **Загрузка архива** с CSS-файлами
- **Переключение** между Default и Custom

## Предусловия

- Права администратора
- Созданные CSS-файлы для кастомного дизайна
- Настройки модуля: `/bitrix/admin/settings.php?lang=ru&mid=ithive.homepage`

## Шаги

### Один сайт

1. Создайте файл `.css`, положите в папку `custom`
2. Сохраните в архив `custom.zip` (структура: `custom.zip > custom > .css`)
3. Настройки модуля → вкладка «Кастомные стили» → загрузите архив
4. Вкладка «Компоненты» → «Шаблон модуля» → «Custom»
5. Для возврата к стандартному виду: выберите «Default»

### Несколько сайтов

1. Создайте отдельный CSS-файл для каждого сайта
2. Положите файлы в разные папки с разными именами (например, `custom` и `custom dep2`)
3. Сохраните в разные архивы (`custom.zip` и `custom dep2.zip`)
4. Загрузите все архивы во вкладке «Кастомные стили»
5. Вкладка «Компоненты» → откройте вкладку нужного сайта → выберите соответствующую папку со стилями
6. Для возврата: выберите «Default»

## Граничные случаи

> Edge cases, ограничения, известные проблемы.

- При частичном изменении стилей архив должен содержать и дефолтные стили модуля
- Дефолтные стили: `/bitrix/css/ithive.homepage/templates/default` (`homepage.css`, `style.css`)
- Рекомендуется изменять стили в скопированных файлах, а не в оригинальных
