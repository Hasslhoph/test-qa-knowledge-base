# Использование модуля

**Модуль:** [[Module - Управление целями. Каскад]]

## Описание

Статья описывает этапы работы с модулем: от подготовки до получения итогового отчета. Включает примеры корпоративных и личных целей, а также их критериев.

## Ключевые функции

> Полный цикл работы с модулем по этапам.

- Определение целей через стратегическую сессию
- Распределение ролей (6 ролей пользователей)
- Заполнение справочника целей
- Создание процессов целеполагания
- Запуск процессов для подразделений
- Заполнение и согласование карточек целей
- Заполнение и согласование карточек оценки
- Подведение итогов через сводный отчет

## Этапы работы

### 1. Определение целей

До начала работы проведите стратегическую сессию для формулировки долгосрочных корпоративных целей. Примеры:
- Диверсификация продаж, выход на иностранные рынки
- Внедрение системы обучения в компании
- Увеличение юзабилити сайта
- Обеспечение объема продаж

### 2. Распределение ролей

6 ролей в модуле:
1. **Сотрудник** — заполняет карточки целей и оценок
2. **Руководитель подразделения** — согласует цели и оценки сотрудников
3. **Функциональный руководитель** — дополнительное согласование
4. **Директор** — согласует цели всех сотрудников в подчинении
5. **Администратор модуля** — полный доступ, финальное утверждение
6. **Сотрудник внутреннего контроля** — просмотр всех данных (только чтение)

### 3. Заполнение справочника целей

- Создать типы целей (только администраторы портала)
- Администратор модуля заполняет каталог целей с настройками: тип, вес, каскадирование, видимость

### 4. Создание процессов

Процесс — срок достижения целей (квартал, полгода, год). Параметры:
- Длительность процесса
- Дни на составление и согласование целей
- Периодичность отчетов (неделя, 2 недели, месяц, квартал, полугодие)
- Дни на составление и согласование отчетов
- Дата отложенного старта
- Дата запуска финальной оценки
- Максимальный процент выполнения

### 5. Запуск процессов

- Определить подразделения для каждой цели
- Привязать процессы к подразделениям через оргструктуру
- Настроить функциональную подчиненность
- Запустить процессы через вкладку «Контроль процессов»

### 6. Заполнение карточек целей

- Карточки создаются автоматически после запуска процесса
- Руководители заполняют первыми (при включенном каскадировании)
- Сотрудники выбирают цели из каталога, прописывают критерии и вес
- Руководители каскадируют цели подчиненным
- Пользователи добавляют индивидуальные цели
- Карточки отправляются на согласование

### 7. Заполнение карточек оценки

- Создаются автоматически после утверждения карточки целей
- Сотрудник указывает процент достижения и комментарий
- Оценки проходят те же этапы согласования
- Процесс повторяется для каждого отчетного периода

### 8. Подведение итогов

- Прогресс из утвержденных карточек попадает в сводный отчет
- Отслеживание средних показателей по корпоративным и индивидуальным целям
- Результаты для разработки финансовых стимулов

## Предусловия

- Модуль установлен и настроен
- Роли распределены
- Справочник целей заполнен
- Процессы созданы

## Граничные случаи

- При включенном каскадировании руководители должны заполнить карточки первыми — подчиненные не смогут начать заполнение пока цели руководителей не утверждены
- На одного пользователя одновременно может быть запущен только 1 процесс
