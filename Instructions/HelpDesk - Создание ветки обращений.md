# HelpDesk - Создание ветки обращений

**Модуль:** [[Module - HelpDesk]]

## Описание

Создание и настройка веток обращений — разделов, в которых хранятся обращения одной тематики. Ветка может быть любой вложенности: от короткой («Другое») до многоуровневой (5-8 шагов). Доступ к созданию/редактированию — только у администраторов портала и администраторов ТП.

## Ключевые функции

### Структура ветки
- **Первый шаг** — стандартный: заголовок + выбор ветки (нельзя изменить)
- **Промежуточные шаги** — подразделы (разделы) с описанием и/или вопросами
- **Финальный шаг** — предпросмотр, редактирование ответов, прикрепление файлов, указание критичности (присутствует по умолчанию)

### Типы вопросов (8 типов)
- **Текстовое поле** — развёрнутый свободный ответ, без ограничения символов
- **Строка** — краткий однострочный ответ
- **Флажок** — чекбокс Да/Нет
- **Опция анонимности** — флажок с символьным кодом `ANONYMOUS_TICKET`; автор не видит обращение после отправки; используется спецпользователь helpdesk.anonymous
- **Выпадающий список** — выбор одного варианта из списка
- **Переключатель** — выбор одного варианта кнопками
- **Несколько полей** — несколько однострочных ответов с подзаголовками
- **Дата и время** — указание конкретной даты/времени
- **Число** — ввод целого числа

### Настройки вопроса
- Активность (скрытые вопросы видны только администраторам портала)
- Название (формулировка вопроса)
- Сортировка (очерёдность на слайдере)
- Обязательность заполнения
- Описание для анонса (подсказки, иллюстрации)
- Привязка к разделам (можно привязать к нескольким)

### Массовые действия с вопросами
- Редактирование, удаление, активация/деактивация
- Перенос в другой раздел
- Добавление привязки к разделу

## Предусловия

- Права администратора портала или администратора ТП
- Путь: `/bitrix/admin/` → Контент → HelpDesk: Категории мастера

## Граничные случаи

- Деактивация раздела скрывает его и все вложенные подразделы/вопросы для всех пользователей, включая администраторов
- Удаление раздела/вопроса не влияет на уже созданные обращения
- Изменения в вопросах/разделах не отражаются на уже созданных обращениях
