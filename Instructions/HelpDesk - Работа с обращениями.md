# HelpDesk - Работа с обращениями

**Модуль:** [[Module - HelpDesk]]

## Описание

Работа с потоком обращений с точки зрения сотрудников техподдержки: просмотр, обработка, редактирование, привязка задач, скрытые комментарии.

## Ключевые функции

### Просмотр обращений
- Уведомление ответственному о новом обращении
- 5 индикаторов: клиент ТП (вы ответственный), клиент ТП (не ответственный), ваш ответ, сотрудник ТП, закрыто

### Обработка обращений

**Смена статуса:**
- Новому обращению — первый статус (по умолчанию «Новое»)
- Клик на нужный отрезок в строке статусов
- При автозаполнении полей — появляются «Дедлайн» и/или «Взяли в работу»

**Редактирование обращения** (слайдер или список):
- Заголовок
- Уровень поддержки (SLA) — при смене применяются настройки нового SLA, ответственный не сбрасывается
- Категория — при смене ответственный не изменяется
- Критичность
- Статус
- Уровень сложности
- Предполагаемая дата закрытия
- Ответственный (кнопка «Сменить»)
- Уведомить автора (галочка)
- Время решения (информационное, нигде не выводится)
- Закрыть при отсутствии ответа через (автозакрытие, дни)

### Привязка задач к обращениям (с v6.0.0)
- Кнопка «Связанные задачи» (если включено в настройках модуля)
- **Создать** — новая задача на основе обращения; название = название обращения; постановщик/исполнитель = текущий пользователь; наблюдатели = все из обращения + автор (если включено); дедлайн по настройкам модуля
- **Привязать** — поиск и привязка существующей задачи
- Клиенты ТП видят блок если имеют доступ хотя бы к одной привязанной задаче
- Сотрудники ТП могут просматривать и откреплять задачи
- Блок несёт информационную функцию — редактирование/закрытие задач не влияет на обращения и наоборот

### Скрытые комментарии
- Доступны пользователям с правами «сотрудник ТП» и выше
- Не видны авторам и наблюдателям (не сотрудникам ТП)
- Не учитываются в колонке «Сообщений» для обычных пользователей
- Выделены розовым цветом для привилегированных пользователей
- Галочка «Скрыть» перед отправкой; настройка запоминается
- Публичный комментарий можно скрыть (только автор); скрытый можно сделать публичным (только автор)

### Удаление обращений
- Доступно администраторам ТП и администраторам портала
- Обращение полностью стирается с портала

### Массовое редактирование
- Доступно сотрудникам ТП и выше
- Только редактирование заголовков

## Предусловия

- Права «сотрудник ТП» и выше
- Функционал связанных задач включён в настройках модуля

## Граничные случаи

- Блок «Связанные задачи» не влияет на статус/закрытие обращений и задач
- Скрытые комментарии недоступны в мобильном приложении
