# HelpDesk - Настройка полей в публичной части

**Модуль:** [[Module - HelpDesk]]

## Описание

Настройка пользовательских (кастомных) полей, выводимых в публичной части модуля — на слайдер обращения, в фильтры и в список обращений.

## Ключевые функции

### Стандартные пользовательские поля
1. **Связанные задачи** — список задач, привязанных к обращению
2. **Предполагаемая дата закрытия** — плановая дата закрытия тикета
3. **Взяли в работу** — дата/время начала работы над обращением (автозаполнение при смене статуса)
4. **Дедлайн** — крайний срок решения (заполняется автором на слайдере создания)
5. **Уведомить автора** — техническое поле, не выводить в публичную часть

### Два типа кастомных полей

**Поля, заполняемые при смене статуса:**
- Типы: «Дата со временем», «Число», «Целое число»
- Создание: Настройки → Настройки продукта → Пользовательские поля → Объект: `SUPPORT`
- Код: префикс `UF_` + латиница
- Автозаполнение: Настройки → Улей: Helpdesk → вкладка «Заполнение полей»
  - Для «Дата со временем»: выбор статуса, при котором заполняется; опция «Перезаписывать»
  - Для «Число»/«Целое число»: статус + два поля с датами для расчёта разницы; опция «Брать по абсолютному значению»

**Поля, заполняемые на основе ответов автора:**
- Привязываются к типам вопросов из мастера обращений
- Соответствие типов: Текстовое поле→Строка, Флажок→Да/Нет, Выпадающий список→Список, Дата→Дата/Дата со временем, Число→Число/Целое число
- Привязка: режим правки → шестерёнка → «Привязка полей мастера к полям обращений/тикетов»

### Вывод полей в публичную часть
- Режим правки → шестерёнка → «Дополнительные настройки»
- «Показывать пользовательские поля сотрудникам ТП» / «Показывать пользовательские поля клиентам ТП»
- Поля появляются в настройке списка и фильтров

## Предусловия

- Права администратора портала

## Граничные случаи

- Для корректной работы поля «Дедлайн» нужна привязка к вопросу типа «Дата и время»
- Тип данных и код поля задаются только при создании
