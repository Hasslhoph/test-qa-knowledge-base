# HelpDesk - Настройка ответственных и наблюдателей

**Модуль:** [[Module - HelpDesk]]

## Описание

Назначение ответственных и наблюдателей за обращениями. Поддерживается ручное и автоматическое назначение. Автоматическое назначение работает по логике: проверка категории → ответственный по категории → единый ответственный для новых → пустое поле.

## Ключевые функции

### Способы назначения ответственных

**Ручное назначение:**
- Через поле «Ответственный» при создании/редактировании обращения
- Через кнопку «Взять в работу» (наблюдатель по категории становится ответственным)

**Автоматическое назначение:**
1. **Единый ответственный для всех новых** — настраивается в модуле «Техподдержка»; применяется если нет ответственного по категории
2. **Ответственные по категориям** — настраивается в настройках HelpDesk; 4 способа:
   - Единый ответственный (один сотрудник для категории)
   - Случайный выбор из группы
   - Единый ответственный в зависимости от подразделения автора
   - Случайный выбор из группы в зависимости от подразделения

### Привязка разделов мастера к категориям
- Публичная часть → режим правки → шестерёнка → «Привязка разделов мастера к категориям»
- Выбирать разделы последнего уровня (где заполняются поля)

### Наблюдатели по категории
- Роль соисполнителя: сотрудник ТП в очереди на ответственность
- Добавляются только автоматически (через группы)
- Кнопка «Взять в работу» доступна только в первом статусе
- Настройка: Настройки → Улей: Helpdesk → вкладка «Наблюдатели по категориям»

### Наблюдатели за обращением
- Ручное добавление: автор — только в свои; сотрудник ТП — в свои или где ответственный; администратор — в любые
- Автоматическое: прямой руководитель автора (настраивается по категориям)

## Предусловия

- Созданы ветки обращений и категории
- Права администратора для настройки ответственных по категориям

## Граничные случаи

- Наблюдатели по категории нельзя добавить вручную или изменить через редактирование
- Кнопка «Взять в работу» пропадает при переходе обращения из начального статуса
- Смена ответственного в созданном обращении — только вручную
