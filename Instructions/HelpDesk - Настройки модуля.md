# HelpDesk - Настройки модуля

**Модуль:** [[Module - HelpDesk]]

## Описание

Основные настройки модуля HelpDesk в административной части портала. Доступ — только у администраторов. Путь: `/bitrix/admin/settings.php?lang=ru&mid=ithive.helpdesk`. Четыре вкладки настроек. Настройки можно выполнить индивидуально для каждого сайта (департамента).

## Ключевые функции

### Вкладка «Настройки»

1. **Доступ к публичной части** — выбор отделов/групп/сотрудников, которые видят модуль и пункт меню. Важно: даже администраторы ТП/портала не видят модуль без доступа в этой настройке
2. **Блок «Ваш уровень обращений»** — показывать информацию об SLA (десктоп и мобильное приложение)
3. **Создавать обращение в статусе** — статус новых обращений (по умолчанию — первый)
4. **Редактирование в первом статусе** — авторы могут редактировать открытые обращения в первом статусе
5. **«Взять в работу» → статус** — статус при нажатии кнопки «Взять в работу»
6. **«Возобновить» → статус** — статус при нажатии кнопки «Возобновить»
7. **Просрочки в списке** — выводить «Просрочено» при превышении времени реакции SLA
8. **Запрет комментариев в закрытом** — запретить автору комментарии в закрытых обращениях
9. **Уведомление при удалении** — уведомлять автора при удалении обращения
10. **Мобильное приложение — дата в списке** — какое поле выводить (предполагаемая дата закрытия / дата создания / дата изменения)
11. **Комментарий при оценке в списке** — выводить колонку с комментарием при оценке
12. **Связь обращений с задачами** — включить/отключить привязку задач к обращениям (с v6.0.0)
13. **Рабочая группа для задач** — группа по умолчанию для задач из обращений (только основной сайт)
14. **Крайний срок через** — авто-расчёт дедлайна задачи из обращения (учитывает «Календарь событий»)
15. **Автор в наблюдатели задачи** — добавлять автора обращения как наблюдателя в созданную задачу

## Предусловия

- Права администратора портала

## Граничные случаи

- Опция рабочей группы для задач работает только для основного сайта
- Доступ к публичной части обязателен для всех ролей, включая администраторов
