# Подходы к внедрению

**Модуль:** [[Module - Корпоративный университет]]

## Описание

Обзор идеального и реальных подходов к внедрению Корпоративного университета в компании.

## Ключевые функции

### Идеальный случай запуска

Одновременное соблюдение условий:
1. Настроены карьерные траектории для всех позиций
2. Заполнены обязательные и рекомендуемые компетенции для всех позиций
3. Для всех компетенций заполнены развивающие действия
4. Проведена оценка имеющихся компетенций у всех сотрудников

### Оценка затрат

- Проработка одной ветки карьерных траекторий (5 позиций, 3 должности): 50-100 человеко-часов
- 10 веток карьерных траекторий: ~750 часов ручного труда
- Использование ИИ (GPT) сокращает затраты минимум в 10 раз

### Реальный подход №1

1. Заполняется схема карьерных траекторий для всех должностей
2. Заполняется система компетенций для подавляющего количества должностей
3. Развивающие действия прорабатываются для одной выделенной ветки

**Преимущества:** Сотрудники из других должностей видят своё положение в схеме и могут самостоятельно развиваться

### Реальный подход №2

1. Схема карьерных траекторий НЕ заполняется
2. Заполняется система компетенций с развивающими действиями для процессов с выраженной потребностью
3. Создаются шаблоны программ (онбординг, развитие до позиции)

**Преимущества:** КУ демонстрирует эффективность в процессах, где компания наиболее нуждается

### Реальный подход №3

Комбинация первых двух подходов.

## Предусловия

- Понимание организационной структуры компании
- Выделение MVP для поэтапного внедрения

## Граничные случаи

- Для больших организаций невозможно сразу создать все элементы КУ
- Решения, генерируемые ИИ, подлежат обязательной проверке компетентными сотрудниками
- Необходимо выделить ограниченный MVP и двигаться поэтапно
