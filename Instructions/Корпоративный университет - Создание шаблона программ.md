# Создание шаблона программ

**Модуль:** [[Module - Корпоративный университет]]

## Описание

Создание шаблонов программ развития и адаптации для упрощения запуска развития.

## Ключевые функции

### Типы шаблонов

- Программа развития
- Программа адаптации

### Создание шаблона

1. Раздел «Университет» → «Шаблоны программ» → «Добавить»
2. Поля:
   - **Название** — понятное и ёмкое (обязательное)
   - **Тип** — Программа адаптации или Программа развития
   - **Срок** — период выполнения (адаптация: 1-3 мес., развитие: 6-24 мес.)
   - **Отправить на согласование руководителю**
   - **Указать целевую должность**
   - **Указать целевую позицию**
   - **Указать следующую программу** — для непрерывного обучения
   - **Указать автоматический запуск** — для новых сотрудников
   - **Компетенции** — с указанием количества дней для каждой

### Автоматический запуск программ

- При активации указать подразделения (обязательное условие)
- Логика определения шаблона:
  - Наследование «отдел со всеми подотделами»
  - При привязке к нескольким подразделениям — программа создаётся для первого
- Автоматизация отрабатывает только для профилей, создаваемых с нуля

### Логика сроков компетенций

- Сроки независимы друг от друга
- Если дата приходится на выходные — перенос на ближайший рабочий день

### Редактирование и удаление

- Раздел «Университет» → «Шаблоны программ» → «Редактировать» / «Удалить»
- При добавлении/удалении компетенций из шаблона — изменения НЕ применяются к уже запущенным карточкам

## Предусловия

- Права: администраторы и группа HR-отдела
- Созданные компетенции

## Граничные случаи

- Если на основе шаблона уже запущены карточки, новые компетенции не добавляются автоматически
- Автоматический запуск работает только для новых сотрудников
