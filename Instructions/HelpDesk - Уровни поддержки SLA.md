# HelpDesk - Уровни поддержки (SLA)

**Модуль:** [[Module - HelpDesk]]

## Описание

SLA (Service Level Agreement) — уровни поддержки для определения приоритетов групп пользователей. Позволяют задать расписание, приоритет, время реакции, доступные категории и справочники для разных групп.

## Ключевые функции

### Настройка расписания
- Путь: Сервисы → Техподдержка → Расписания
- Вкладка «Расписание»: Имя, Описание
- Вкладка «Часы»: для каждого дня — 24 часа / Не работает / Конкретные часы (несколько интервалов)
- Без расписания сохранить SLA невозможно

### Настройка исключений
- Путь: Сервисы → Техподдержка → Расписания → Исключения
- Для учёта праздничных дней и переносов
- Поля: Имя, Действие (отдых/работа), Период, Уровни поддержки

### Настройка уровней поддержки
- Путь: Сервисы → Техподдержка → Уровни поддержки
- Вкладка «Уровень техподдержки»:
  - Название, Сайты
  - Приоритет (число: чем больше, тем выше; при равном — меньший ID)
  - Время реакции (дни/часы/минуты) — при превышении появляется «Просрочено»
  - Напоминание по Email до окончания времени реакции
  - Ответственный по умолчанию (если не настроены ответственные по категориям)
  - Расписание
  - Считать крайний срок от: даты создания или последнего ответа клиента
- Вкладка «Группы» — выбор групп пользователей для данного SLA
- Вкладка «Доступ» — какие категории, критичности, статусы, сложности и оценки доступны

## Предусловия

- Права администратора портала или администратора ТП
- Создано расписание работы техподдержки

## Граничные случаи

- При редактировании SLA изменения применяются к новым и существующим обращениям
- Удалить можно только SLA, в рамках которых не создано ни одного обращения
- При изменении часов работы модуль автоматически пересчитывает время реакции во всех обращениях
- Если у группы доступ к нескольким SLA — применяется SLA с высшим приоритетом
