# HelpDesk - Создание обращения и мониторинг прогресса

**Модуль:** [[Module - HelpDesk]]

## Описание

Процесс создания обращений для разных ролей пользователей и мониторинг прогресса по обращениям.

## Ключевые функции

### Создание обращения
- Кнопка в правом верхнем углу страницы HelpDesk
- Первый шаг: заголовок + выбор ветки
- Промежуточные шаги: зависят от ветки
- Финальный шаг: предпросмотр, редактирование ответов, прикрепление файлов, критичность
  - Заголовок обращения
  - «Ваш вопрос» — ответы на все вопросы + визуальный редактор + вставка скриншотов (Ctrl+V)
  - Файлы — .docx, .zip, .xls, .pdf, .jpg, .png; лимит по умолчанию 10000 Кб
  - Критичность — срочность/важность

### Список обращений
- Обычные пользователи видят свои тикеты + тикеты где наблюдатели
- Анонимные обращения не видны автору
- Индикаторы: ответ ТП (красный), ваш ответ, закрыто

### Слайдер обращения

**Центральный блок:**
- Содержание обращения
- Пользователи, просматривавшие за последние 5 минут
- Комментарии

**Правый блок (техническая информация):**
- Текущий статус и дата
- Срок закрытия (автоматический расчёт)
- Создано / Изменено
- Уровень сложности
- Критичность
- Категория
- Статус поддержки (SLA)
- Статус обращения
- Предполагаемая дата закрытия
- Взяли в работу
- Источник/От кого
- Ответственный
- Наблюдатели по категории
- Наблюдатели

### Редактирование обращения автором
- Доступно если: не закрыто + первый статус + опция включена в настройках
- Поля: заголовок, критичность, содержание, файлы

### Мониторинг
- Индикатор и счётчик в левом меню
- Уведомления о всех действиях
- Комментарии
- Смена статуса
- Связанные задачи (если автор имеет доступ к задаче)

### Привилегированные пользователи как авторы
- Сотрудники ТП: такой же доступ к своим обращениям, но не могут редактировать свои и менять статус
- Администраторы ТП: полный доступ к своим обращениям

## Предусловия

- Права «клиент ТП» и выше
- Созданы ветки обращений

## Граничные случаи

- Анонимные обращения не отображаются у автора в списке
- Привилегированные пользователи не являются клиентами ТП даже при создании обращений
