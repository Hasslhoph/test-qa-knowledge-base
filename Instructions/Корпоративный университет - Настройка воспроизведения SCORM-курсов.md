# Настройка воспроизведения SCORM-курсов

**Модуль:** [[Module - Корпоративный университет]]

## Описание

Решение проблемы скачивания SCORM-курсов вместо воспроизведения на портале.

## Ключевые функции

### Причина проблемы

- Битрикс24 запретил воспроизведение HTML-файлов из папки `/upload/` в целях безопасности
- SCORM-курсы загружаются в `/upload/` → браузер скачивает файл вместо отображения

### Способ №1: Создание новой папки

1. Административная часть → «Контент» → «Структура сайта» → «Файлы и папки»
2. Создать папку `upload2`
3. Копировать `/upload/learning/` в `/upload2/`
4. Сервисы → «Обучение» → «Курсы» → открыть SCORM-курс
5. Вкладка «Содержание» → заменить `/upload/` на `/upload2/` в пути к файлу
6. Повторять для каждого нового SCORM-курса

### Способ №2: Изменение настроек сервера

- Обратиться к системному администратору
- В конфигурации Nginx удалить `html` из фильтра:

```nginx
location ~* ^/upload/.*([^/]+)\.(html|htm|php|php3|php4|php5|php6|phtml|pl|asp|aspx|cgi|dll|exe|shtm|shtml|fcg|fcgi|fpl|asmx|pht)$ {
    add_header Content-Disposition "attachment";
}
```

## Предусловия

- Права на способ №1: администраторы
- Права на способ №2: системный администратор сервера
