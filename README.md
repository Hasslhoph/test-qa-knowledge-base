# QA База знаний — iThive

> Полноценная база знаний для QA-тестирования модулей платформы **iThive**, работающая в связке **Obsidian** + **opencode** с AI-субагентами.

---

## Для чего это нужно

Централизованная база знаний для QA-команды, которая тестирует модули платформы iThive.

**Проблема:** при тестировании 16+ модулей QA-инженерам нужно быстро находить информацию о настройке, функциональности и связях между модулями.

**Решение:** база знаний в Obsidian с wikilinks, подключённая к AI-ассистенту opencode. AI читает все инструкции и может мгновенно отвечать на вопросы, создавать чек-листы проверок и индексировать новые документы.

---

┌───────────────────┐  ┌──────────────────┐  ┌──────────────────┐

└─────────┬─────────┘  └────────┬─────────┘  └────────┬─────────┘
          │                      │                     │
          ▼                      ▼                     ▼
┌─────────────────────────────────────────────────────────────┐
│                   Obsidian Vault                            │
│                                                             │
│  Instructions/  ← 200+ инструкций по всем модулям           │
│  Modules/       ← 16 файлов модулей со связями              │

│  MOC-файлы      ← Навигация и карты документов              │

### Ключевые возможности

- **Мгновенные ответы** — AI читает всю базу знаний и отвечает на вопросы за секунды
- **Связанность** — wikilinks связывают модули, инструкции и тест-кейсы между собой
- **Автоматическая индексация** — новые инструкции добавляются в базу автоматически
- **Генерация чек-листов** — AI создаёт детальные чек-листы с техниками тест-дизайна
- **Бесплатно** — работает с бесплатной моделью Qwen 3.6 Plus Free

## 🤖 AI-субагенты

В системе работают три специализированных субагента. Каждый решает свою задачу и не мешает другим.

### @obsidian-indexer — Индексатор инструкций

**Что делает:**
- Принимает текст инструкции и анализирует её содержимое
- Автоматически определяет к какому модулю относится инструкция
- Создаёт файл в папке `Instructions/` с правильным форматом
- Добавляет wikilinks на связанные модули и документы
- Проставляет теги (`#module/{slug}`, `#type/instruction`)
- Заполняет frontmatter (properties) с метаданными
- Обновляет MOC-файлы (карты документов)
- Обновляет файлы модулей — добавляет ссылку на новую инструкцию

**Что НЕ делает:**
- Не создаёт тест-кейсы и чек-листы (это задача @qaithive)
- Не удаляет существующие файлы

**Когда вызывать:**
```
@obsidian-indexer [вставь текст новой инструкции]
```

---

### @qa-teacher — Справочник по модулям

**Что делает:**
- Отвечает на вопросы по функциональности модулей
- Даёт пошаговые инструкции по настройке
- Объясняет как работают отдельные функции
- Показывает связи между модулями
- Приводит примеры из базы знаний
- Ссылается на конкретные документы-источники

**Что НЕ делает:**
- Не создаёт, не редактирует и не удаляет файлы
- Не запускает команды в терминале
- Только читает базу знаний и отвечает

**Когда вызывать:**
```
@qa-teacher как настроить уведомления в HelpDesk?
@qa-teacher как покрасить фон виджета через брендирование?
@qa-teacher какие модули зависят от мультиязычности?
```

---

### @qaithive — Генератор чек-листов

**Что делает:**
- Анализирует условие задачи (новая фича, баг-фикс, регрессия)
- Изучает базу знаний — инструкции модуля, связи, MOC-файлы
- Создаёт детальный чек-лист с техниками тест-дизайна:
  - **Позитивные проверки** — основной сценарий (Happy Path)
  - **Негативные проверки** — невалидные данные, ошибки доступа
  - **Граничные значения** — мин/макс, пустые поля, спецсимволы
  - **Интеграционные проверки** — связанные модули, API
  - **UI/UX проверки** — отображение, адаптивность, валидация
  - **Регрессия** — существующий функционал модуля
- Сохраняет чек-лист в `Test Cases/` (по запросу)

**Техники тест-дизайна:**
- Классы эквивалентности
- Граничные значения
- Таблица решений
- Попарное тестирование
- State Transition

**Когда вызывать:**
```
@qaithive задача: добавить экспорт тикетов в CSV в HelpDesk. Создай чек-лист.
@qaithive задача: исправлена ошибка авторизации в Корпоративном университете. Чек-лист регрессии.
```
